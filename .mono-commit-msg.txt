Stop Enza branding leaking onto the shared login; add /enzaglobalmedia vanity path (ASCII-only)

The regular login was showing Enza branding because resolvePublicBrandByHost fell back to the platform theme (which had picked up Enza colours/logo). Changed it: a partner brand only applies to a PUBLIC page on that partner's own custom domain; on the app's own domain it now returns the neutral Synops default. So the regular sign-in has no Enza branding again, while the Enza sign-in stays branded because its links carry ?p=enza-global. Also added a clean /enzaglobalmedia route that renders the Enza landing (alongside /p/:slug), and the admin Landing button opens /enzaglobalmedia for Enza.
