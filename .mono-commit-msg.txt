Accounts: fix org dropdown, real partner provisioning, cohort self-enrol links, real View-as impersonation (ASCII-only)

Four fixes to Accounts & Roles / delivery. (1) Create & Invite org dropdown was empty for super admins (it double-filtered by active partner); it now lists every org the backend already scopes to. (2) Send invite is now REAL: POST /partners/:partnerId/members (partner_admin or super_admin) creates a Coach, Org-admin or Learner in a chosen org as invited and returns a one-time set-password link (emailed if mail is on, else shown to copy/share). (3) Learner self-enrol via a shareable cohort link: new routes/enrol.ts with POST /classes/:id/join-link (admin) and PUBLIC GET/POST /enrol/:code - a new learner registers, sets a password, is enrolled into the cohort's courses and logged in, all from a link (self-creating class_join_codes table, no migration). New public page /join/:code plus a Share join link + Share on WhatsApp button on the class detail page. (4) Real View-as impersonation for partners: POST /partners/:partnerId/impersonate/:userId mints a real short-lived session (like the super-admin path) scoped to the partner's own accounts; the account drawer gains a View as button that drops you into the learner's real course view with the existing stop-impersonating banner.
