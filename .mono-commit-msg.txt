Partner Hub convergence stage 1: seed real Enza hub data (ASCII-only)

The Financial/Funders/Documents/Accounts hub pages already query real DB tables by partner_id, but those tables were empty for the live Enza partner, so every hub read as empty (the demo figures lived only in a front-end mock). New seedEnzaHub() creates real records for the live Enza partner + its orgs: billing subscriptions + invoices (paid + due), MICT SETA + foundation funding agreements, filed partner documents (MSA, funding agreement, POPIA addendum, Q1 report), and a delegated admin per org. Idempotent; creates the hub tables IF NOT EXISTS first. Exposed as POST /platform/seed-enza-hub (super admin) and a 'Seed Hub Data' button in the gated Internal tools panel. After seeding, the four hub pages populate with real numbers. (Stage 2 will point the Organisations overview at these same endpoints and retire the mock layer.)
