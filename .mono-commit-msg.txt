Swap CVE-affected file parsers: xlsx->exceljs, pdf-parse->unpdf (ASCII-only)

Replaced two vulnerable, upload-path parsers. SheetJS/xlsx is stuck on 0.18.5 on npm (unpatchable prototype-pollution CVE-2023-30533 + ReDoS CVE-2024-22363); pdf-parse is unmaintained since 2018 and bundles an old pdf.js. Both parse attacker-controlled uploads. Migrated all three usages to maintained libs: gradebookExport + accreditationExport (workbook writing) and extractText (spreadsheet reading) now use exceljs; extractText PDF reading uses unpdf (getDocumentProxy + extractText). Removed xlsx + pdf-parse from package.json (exceljs already added, unpdf added); updated build.mjs external list to exceljs/unpdf. Railway + CI install with --no-frozen-lockfile so they resolve fresh.

VALIDATION: esbuild clean on gradebookExport.ts, extractText.ts, accreditationExport.ts; no xlsx/pdf-parse imports remain.
