Fix readings white screen: React.Fragment used as a value but React is only named-imported (ASCII-only)

The Readings reader crashed to a blank screen because MarkdownView rendered <React.Fragment>, but ModuleViewer imports React with named imports only (no default React value), so React.Fragment was undefined at runtime. Replaced it with a keyed <span>. Readings now render their Markdown as expected.

VALIDATION: esbuild clean on ModuleViewer.tsx; grep confirms no other React.<value> uses.
