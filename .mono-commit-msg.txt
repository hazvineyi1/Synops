Heal assignments table drift so all 15 Enza courses seed + assign (ASCII-only)

The Provision toast named the real culprit: insert into assignments failed because the production table predates several columns (assignment_type, due_date, available_from/until, points_possible, allow_late_submissions, late_penalty_percent, etc.) and CREATE TABLE IF NOT EXISTS never backfills them - so every course died on its final insert. Added healAssignmentsTable() (ALTER TABLE assignments ADD COLUMN IF NOT EXISTS for all schema columns) run at the start of ensureEnzaCourses. Also backfills the applied-project assignment on the orphan courses earlier runs created before the heal (they have every other piece). Now Provision Enza Global assigns all 15 and Seed Enza Cohort enrols learners into their courses.

VALIDATION: esbuild clean on enzaSeed.ts, platform.ts.
