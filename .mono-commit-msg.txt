Fix Create-partner slug editing + add logo upload to branding edit (ASCII-only)

SLUG BUG: the tenant-slug field ignored deletions and appended to the name-derived value ('test-partner-inc' + 'hello' = 'test-partner-inchello'). Cause: the displayed slug fell back to slugify(name) whenever the edited field was empty (form.slugTouched && form.slug), so clearing it snapped back to the name slug and every keystroke re-appended. Now once touched the field shows exactly what was typed (even empty); a separate finalSlug = slugify(slug) is what gets submitted/validated. The per-keystroke sanitiser also no longer strips trailing hyphens, so multi-word custom slugs can actually be typed. A custom slug that differs from the name now works.

BRANDING PARITY: the Configure -> Branding edit panel only had a raw 'Logo URL' text box while Create-partner had an upload button. Added an 'Upload logo' file input to the branding panel (reuses readImage, stores a data URL) so both flows can upload, not only paste a URL.
