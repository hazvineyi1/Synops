Perf: paginate org-members + partner-audit endpoints (limit/offset/total)

Extends the partner-roster pagination pattern to the remaining capped lists so nothing can silently truncate. GET /organisations/:orgId/members now supports ?limit (default 2000, max 2000), ?offset and ?search (case-insensitive over name + email) with stable ordering (updated_at desc, id) and the true count in X-Total-Count. GET /partners/:id/audit gains ?offset + X-Total-Count (limit already capped at 500). Both keep their plain-array bodies, so every existing consumer is unaffected. Frontend: the org Members page gains a search box (role-count stats stay accurate because they read the full list). Files: routes/organisations.ts, routes/partners.ts, pages/OrgMembers.tsx.
