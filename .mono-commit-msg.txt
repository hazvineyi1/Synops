Fix unenrolled course modules 403 hang: allow catalogue view + graceful error UI (ASCII-only)

Browsing an unenrolled catalogue course and opening the Modules tab hung on gray skeletons forever. Two causes: (1) GET /courses/:id/modules was enrolment-gated (participantOn), so a catalogue viewer got 403; (2) the CourseDetail Modules tab rendered its skeleton on '!modules', which stays undefined on error, so a 403 span the skeleton indefinitely.

BACKEND: new shared scope helper canViewCourseCatalog (tenant-owned OR partner-assigned OR platform OR hub/enrolled) now gates the module LIST (titles/order only - catalogue overview info); actual content (beats/readings/cases) stays enrolment-gated. Refactored the course-detail route to use the same helper.

FRONTEND: the Modules tab now distinguishes loading vs error (retry:false) and shows 'Enrol in this course to view its modules' on a genuine 403 instead of an infinite skeleton.

VALIDATION: esbuild clean on scope.ts, modules.ts, courses.ts, CourseDetail.tsx.
