Fix Studio Edit white-screen crash (React error 31 on object-valued beat fields) (ASCII-only)

StudioEdit blanked to a white screen whenever a draft had a beat whose scenario (or other field) was a structured object (e.g. an AI-generated quiz object with keys situation/question/options/correctOption/explanation) instead of plain text - rendering an object as a React child throws React error 31 and takes down the whole Edit + Publish page. Added toBeatText() which coerces any beat field to a readable string (formatting quiz/scenario objects into situation + question + labelled options + explanation), and routed every rendered/edited beat field (title, narration, bullet points, scenario) and the form-state init through it. Studio generate -> edit -> publish-to-course now works end to end (backend was already fine).

VALIDATION: esbuild bundle clean on StudioEdit.tsx.
