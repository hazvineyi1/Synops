Fix Seed Hub Data: heal billing_subscriptions column drift + call ensureHubTables

seedEnzaHub read from billing_subscriptions with a typed SELECT before the table was created/healed, and ensureHubTables (which only did CREATE TABLE IF NOT EXISTS) was never called. On a pre-existing table with an older column set the SELECT threw 'column does not exist'. Now ensureHubTables also runs ALTER TABLE ADD COLUMN IF NOT EXISTS for every column of all five hub tables (billing_subscriptions, billing_invoices, funding_agreements, partner_documents, delegated_admins), all NOT NULL cols carry defaults, and it is invoked at the start of seedEnzaHub before any read/write.
