Fix partner Overview stuck on an infinite loading skeleton (ASCII-only)

The partner Overview hung on grey loading cards forever. Root cause: a partner_admin whose partnerId was null (created via a provisioning path that only set organisationId). The dashboard keys the partner-stats query off partnerId, and the generated hook DISABLES the query when partnerId is null - so the data never arrived and the 'if (!stats) return skeleton' guard never cleared.

Two-part fix. Backend: /auth/me now heals a facilitator-tier account missing partnerId by backfilling it from the owning organisation (and persisting it), so the query enables and real numbers render. Frontend: PartnerAdminDashboard and OrgAdminDashboard no longer gate the whole page on '!data' - they use isLoading and fall back to zeros/orgs list once the query settles, so the Overview can never hang on a skeleton again regardless of role or query error.

VALIDATION: esbuild clean on auth.ts and Dashboard.tsx.
