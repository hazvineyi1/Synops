Praxis CI + nightly DB backup coverage (ASCII-only)

The existing GitHub Actions only covered Coach: db-backup.yml dumped only the Coach Postgres, and ci.yml built only the Coach (paideia) deployment - so a Praxis-breaking change could pass CI and Praxis's own database had no backup. Added: (1) a backup-praxis job in db-backup.yml that pg_dumps the Praxis Postgres to a 90-day artifact using a new PRAXIS_DATABASE_PUBLIC_URL secret, and SKIPS gracefully (warning, not failure) until that secret is set so the Coach backup is never blocked; (2) a build-praxis job in ci.yml that reproduces Dockerfile.praxis (install + build the self-contained praxis/ workspace's SPA + API) so Praxis build breakage fails in CI instead of in the Railway image build.

VALIDATION: both workflow YAMLs parse; jobs [backup, backup-praxis] and [build, build-praxis].
