Real user management: absolute reset links + Create user in Platform Console (ASCII-only)

Two blockers fixed:
1) Reset/set-password links were relative (/reset-password?token=...) because APP_URL was unset, so copying them did not work -> could not set a password. platform.ts now builds an ABSOLUTE base from the request host (appBase(): APP_URL if set, else req.protocol+host; trust proxy makes https correct on Railway). Applied to the reset-link endpoint.
2) No way to add users from the console (users could only be created against an org). New POST /platform/users (super_admin): create a user by email/name/role (any of the 7 roles), status invited, and returns a one-time set-password link in the same response. Frontend: a Create user button + dialog on the Platform Console Users tab that shows the copyable link.

So: create user -> copy link -> they set a password -> sign in. Set APP_URL on Railway for tidy links (optional now that it falls back to the request host).

VALIDATION: esbuild clean on platform.ts, platformApi.ts, PlatformConsole.tsx.
