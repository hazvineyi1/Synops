Fix ModuleViewer white-screen crash (React #31 on object-valued beat fields) (ASCII-only)

Opening a module blanked/froze the page whenever a beat had a scenario (or other field) stored as a structured object (e.g. an AI/seeded quiz-scenario with keys situation/question/options/correctOption/explanation) rather than plain text - rendering an object as a React child throws React error 31 and takes down the whole module. Same class as the earlier Studio fix. Added toBeatText() to ModuleViewer and routed every rendered beat field (narration, scenario, points, compare items) through it so object fields render as readable text.

VALIDATION: esbuild bundle clean on ModuleViewer.tsx.
