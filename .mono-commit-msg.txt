Hardening: members pagination + observability (version endpoint + request-id)

(Perf) GET /partners/:id/members now supports ?limit (default 2000 = unchanged, max 2000), ?offset, and ?search (case-insensitive over name + email), with stable ordering (updated_at desc, id) and the true row count in an X-Total-Count header. No-param callers get identical behaviour, so the other pages reading the array (Comms/Funders/Audit/Organisations/OrgHub) are untouched. Frontend: new apiFetchMeta helper reads the header; the Accounts roster gains a debounced server-side search and a Showing N of total + Load more control (500 at a time), so a large partner is never silently truncated. (Observability) New /version endpoint (live commit, deployment id, uptime, node, env) for probes + rollback verification; pino-http now issues one correlation id per request, honouring and echoing X-Request-Id so a client 500 maps to the exact server log line. Files: routes/partners.ts, routes/health.ts, app.ts, praxis/src/lib/api.ts, pages/partner/PartnerAccounts.tsx.
