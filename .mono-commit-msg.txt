Coaching and sections: distinguish loading/error/empty for the course list (ASCII-only)

The Coaching and sections page showed 'Create a course first' whenever no course was selected - which also happened while /courses was still loading OR if it errored, so a slow/failed request masqueraded as 'no courses exist' (the same failure-to-handle-error pattern as the modules hang). Now it shows 'Loading courses...' while fetching, 'Could not load courses. Please refresh.' on error (retry:false), and only 'No courses yet. Create a course first.' when genuinely loaded-and-empty. The Course dropdown shows 'Loading...' instead of 'No courses' during the fetch.
