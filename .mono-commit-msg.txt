Real game assignments: one quiz per course, the rest sequence puzzle, match-up and Jeopardy (ASCII-only)

Assignments are no longer all quizzes. enzaEnrich now rotates a different auto-graded GAME per module (mi%4): 0 = quiz, 1 = Sequence Puzzle (order the steps), 2 = Match-Up (pair idea to example), 3 = Enza Jeopardy (pick a tile, answer, bank points). So a 4-module course has exactly one quiz and three games. Backend: generalized the auto-grader to maybeAutoGradeGame + gradeGame, scoring quiz/order/match/jeopardy/spot/levels server-side from the config answer key and grading via applyGrade (still fires the off-track + gap-plan + coach chain). Frontend: new OrderGame, MatchGame and JeopardyGame components (interactive, animated) plus a shared GameResult tier/XP card; wired into AssignmentDetail's subType rendering, submission body and validation. Re-running Build Full Courses upgrades every assignment to its game.

VALIDATION: esbuild clean on assignments.ts, enzaEnrich.ts, AssignmentDetail.tsx.
