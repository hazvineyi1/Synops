Security: opt-in TOTP two-factor auth (backend) - enrol/verify + login challenge

Adds admin-facing two-factor auth, backend only for now (enrolment + login UI is the next commit). New dependency-free RFC 6238 TOTP lib (base32 + HMAC-SHA1 + skew window + otpauth URI + hashed one-time backup codes) with 17 unit tests, including the official RFC 6238 SHA1 test vectors. Users gain mfa_enabled / mfa_secret / mfa_backup_codes (Drizzle schema + additive boot heal). New self-service endpoints (requireAuth): GET /auth/mfa/status, POST /auth/mfa/setup|enable|disable - a user can only ever manage their own 2FA; backup codes are stored as sha256 hashes and shown once. Login gains a DORMANT second factor: it is skipped entirely unless mfaEnabled is true, so every existing login is unchanged; an enrolled user gets { mfaRequired:true } then re-POSTs email+password+code (TOTP or a one-time backup code). 51 unit tests green. Files: lib/totp.ts(+test), schema/users.ts, lib/dbHardening.ts, routes/auth.ts.
