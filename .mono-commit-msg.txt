Fix root cause of Enza seed failure: heal org_classes column drift (ASCII-only)

The toast revealed the real error: the select on org_classes failed because the production table was created by an older deploy and is missing partner_id / created_by (CREATE TABLE IF NOT EXISTS never adds columns to a pre-existing table, so every Drizzle query that selects all columns fails). That is why the cohort could never seed and the learner logins never got created. Added healClassTables() - CREATE TABLE IF NOT EXISTS plus ALTER TABLE ADD COLUMN IF NOT EXISTS for org_classes (partner_id, created_by, created_at) and the org_class_* tables - run at the very start of seedEnzaCohort before any class access. With the drift healed, the previously added guarantees kick in: accounts ensured + verified, and the toast now reports Logins N/N verified.

VALIDATION: esbuild clean on enzaCohortSeed.ts.
