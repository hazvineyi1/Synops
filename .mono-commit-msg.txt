Praxis ops hardening: readiness probe + optional Sentry (ASCII-only)

Praxis already had helmet, rate limiting, pino logging, CORS and a central error handler. Filled the two Phase-0 gaps: (1) /api/readyz - a real readiness probe that pings the DB with a 3s timeout and returns 503 fast if the DB is unreachable (liveness /healthz stays DB-free so a brief DB blip can't restart the process); (2) optional Sentry error monitoring via lib/observability - dependency-OPTIONAL by design: it only activates when SENTRY_DSN is set AND @sentry/node resolves, never throws from a capture path, and is a clean no-op otherwise, so the service runs identically with or without it. The central 500 handler now reports to Sentry (no-op unless configured). Did NOT add @sentry/node to package.json to avoid a frozen-lockfile install break; enabling Sentry is a deliberate 'pnpm add @sentry/node -F @workspace/api-server' + set SENTRY_DSN.

VALIDATION: esbuild clean on health.ts, observability.ts, app.ts, index.ts.
