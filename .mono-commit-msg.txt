Hardening: perf indexes for hot lookups + demo-login host lockdown

(Perf) dbHardening now creates CREATE INDEX IF NOT EXISTS on the hottest paths at boot: enrolments(course_id), enrolments(user_id), gradebook_items(course_id), gradebook_cells(item_id), org_class_courses(class_id), org_class_learners(class_id), org_classes(org_id), users(partner_id), users(organisation_id). Idempotent, cheap once present, big win at scale. (Security) /auth/demo-login gains an optional DEMO_LOGIN_HOSTS allowlist - when set (comma-separated), the one-click demo only works on those hosts (e.g. the Enza demo site) and 404s everywhere else, so it can never be used to enter a real production tenant. Unset = backward compatible; prod tenants should also set ENABLE_DEMO_LOGIN=0. Files: lib/dbHardening.ts, routes/auth.ts.
