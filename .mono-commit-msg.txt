Configurable gradebook P2: per-item config editor + per-org overrides

Admins/coaches configure how each deliverable is graded, at course level OR per organisation. New PUT /courses/:courseId/gradebook/config upserts per-column config by (sourceType, sourceId): grade_type (points|pass_fail|completion), itemType (counts vs practice), pointsPossible, includeInGrade. With no orgId it writes the course default (gradebook_items, creating an override row for default assignment columns that have none); with an orgId it writes a per-org override in the new gradebook_org_overrides table (course, org, sourceType, sourceId; null fields inherit), created + indexed at boot in dbHardening. Engine getCourseColumns(courseId, orgId?) applies an org's overrides on top of the course default; the learner /me and staff /learner/:id resolve against the learner's own organisation, and the matrix accepts ?orgId. New GradebookConfigDialog with an 'Applies to' scope selector (Course default | <org> override), grouped per category, opened via a 'Configure grading' button on the coach gradebook; the org courses table gets a 'Gradebook ->' link so it is reachable from the admin org pages. POST/PATCH gradebook-items also accept grade_type. Files: schema/gradebook.ts, dbHardening.ts, gradebookEngine.ts, routes/gradebook.ts, gradebookApi.ts, GradebookConfigDialog.tsx, CourseGradebook.tsx, PartnerOrgHub.tsx.
