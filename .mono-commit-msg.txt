Security: MFA enrolment + login UI - two-factor now usable end to end

Frontend for the TOTP backend shipped last commit. New /security page (linked in the sidebar) lets a user turn on two-factor: begin setup, add the shown setup key to an authenticator app (manual entry, no new dependency), confirm a 6-digit code, then save 10 one-time backup codes; and turn 2FA off again with a current code. SignIn gains a dormant second step: signIn(email,password,code?) now returns { mfaRequired } and, when the server challenges an enrolled account, the form reveals a code field (accepts a backup code too). Non-enrolled logins are completely unchanged. Moves the MFA readiness item to done. Files: pages/AccountSecurity.tsx (new), App.tsx, components/layout/AppLayout.tsx, context/SessionContext.tsx, pages/SignIn.tsx.
