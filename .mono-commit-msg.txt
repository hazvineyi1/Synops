Delete user + email delivery for set-password links (ASCII-only)

1) DELETE user: new DELETE /platform/users/:id (super_admin, cannot delete self) hard-deletes the user plus their sessions, password resets, login events, enrolments and section memberships in one transaction; authored content is left. A Delete button is added to the Platform Console user drawer (with confirm). platformApi.deleteUser added.

2) EMAIL delivery: new lib/email.ts (Resend HTTP API, no dep - same pattern as supabaseStorage). When RESEND_API_KEY + EMAIL_FROM are set, Create user, the console Reset link, and self-service forgot-password now EMAIL the set-password link automatically (and the console still shows a copyable link as fallback + an 'emailed' indicator). Unconfigured = no-op, existing manual-link flow unchanged.

ENV (optional, to enable auto-send): RESEND_API_KEY + EMAIL_FROM (a verified sender). Also set APP_URL for tidy links.

VALIDATION: esbuild clean on email.ts, platform.ts, auth.ts, platformApi.ts, PlatformConsole.tsx.
