Hardening: health-gated deploys + input validation + demo-login safe-by-default

(A/Infra) Railway healthcheck now gates on /api/readyz (DB-gated) instead of /healthz, so a deploy is only promoted once it can reach the database. (B/Data) New reusable validateBody helper (typed, lenient - only checks named fields, unknown fields pass through) applied to core write endpoints: POST /organisations, POST /organisations/:orgId/members, POST /partners - malformed payloads now get a clean 400 before hitting the DB; 9 unit tests. Also confirmed the synthetic/placeholder data sweep is complete (no fabricated data left in write routes). (C/Security) Demo login is safe-by-default: 404 on every host except an allow-listed demo host (defaults to enza.synops-consulting.com), configurable via DEMO_LOGIN_HOSTS, hard-off with ENABLE_DEMO_LOGIN=0 - a new tenant on a new domain can never expose it. 34 unit tests green (gradebook 17 + enrolment 8 + validation 9).
