Fix: Stop impersonating restores admin session instead of signing out (retry build)

The impersonation banner's Stop impersonating button called handleSignOut, which revoked the session and dropped the admin at /sign-in. The backend already stashes the admin's own session in the praxis_impersonator cookie and exposes POST /platform/stop-impersonating to restore it. Added SessionContext.stopImpersonating (POST /platform/stop-impersonating then full reload to /dashboard) and wired the banner button to it. Now exiting learner impersonation returns the admin to their own dashboard. Frontend-only. (Prior build of this change failed at the image step with no logs, likely transient; re-triggering.)
