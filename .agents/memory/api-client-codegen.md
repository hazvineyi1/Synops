---
name: Frontend API client is hand-authored
description: How the typed React Query client is generated and when to bypass it.
---

The frontend client in `lib/api-client-react` is generated by Orval from a **hand-authored** `lib/api-spec/openapi.yaml` (run `pnpm --filter @workspace/api-spec run codegen`). It is not auto-derived from Express routes.

**Why:** adding an endpoint to the typed client means editing the YAML spec then regenerating — meaningful overhead for small/internal surfaces.

**How to apply:** for core product endpoints, update `openapi.yaml` + regenerate so the app gets typed hooks. For one-off/internal surfaces (e.g. the admin dashboard), calling the API with direct `fetch` to `/api/...` with `credentials: "include"` is acceptable — session cookies are same-origin so auth carries automatically, matching how the generated custom-fetch behaves on web.
